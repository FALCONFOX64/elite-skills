# Elite Technical Writing Skill

## Core Principles
- **Audience First**: Always identify and adapt to the target audience (beginner, expert, mixed). Use appropriate terminology, depth, and examples.
- **Clarity and Precision**: Use simple, direct language. Avoid jargon unless defined. One idea per sentence when possible. Active voice preferred.
- **Structure and Scannability**: Employ headings, subheadings, numbered/bulleted lists, tables, code blocks, warnings/notes. Follow logical flow: introduction, body, conclusion/summary.
- **Accuracy and Completeness**: Base content on provided facts/sources. Flag assumptions. Include prerequisites, steps, expected results, troubleshooting, limitations.
- **Consistency**: Maintain uniform style, terminology, formatting. Use defined glossaries.
- **Professional Tone**: Objective, authoritative, helpful. No fluff or marketing unless specified.

## Best Practices
1. **Planning**:
   - Clarify purpose, audience, scope.
   - Outline structure (TOC if long).
   - Gather all source material.

2. **Writing**:
   - Start with overviews/summaries.
   - Break complex topics into chunks.
   - Use examples, diagrams (describe if needed), code snippets.
   - Include cross-references, indices where relevant.

3. **Formatting**:
   - Markdown for docs: # headings, **bold**, `code`, ```blocks```, tables.
   - For docs: consistent fonts, spacing, version numbers, dates.
   - Visuals: Suggest or describe images, diagrams, screenshots.

4. **Editing and Review**:
   - Revise for clarity, grammar, flow.
   - Check for ambiguity, contradictions.
   - Read aloud mentally for natural flow.
   - Version control and change tracking.

5. **Specialized Outputs**:
   - **User Manuals**: Prerequisites, step-by-step (one action/step), screenshots references, troubleshooting.
   - **API Docs**: Endpoints, methods, params (required/optional), request/response examples, auth, errors, rate limits.
   - **Reports/Whitepapers**: Executive summary, methodology, findings, recommendations, appendices.
   - **Code Comments/Docs**: Self-documenting code + external docs (e.g., README, architecture).

## When to Ask for Clarification
- Missing audience details, source material, or specific style guide (e.g., Google, Microsoft).
- Ambiguous requirements or technical details.

## Integration with Other Skills

## Examples
**Prompt Response Structure**:
- Confirm understanding.
- Provide draft.
- Offer revisions or expansions.
- Suggest improvements.

Follow these for every technical writing task to deliver elite-quality results.

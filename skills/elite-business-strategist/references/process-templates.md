# Process Templates for Idea-to-Execution

Use these as optional scaffolds when correlating ideas to real processes. Adapt heavily; do not force-fit.

## 1. Core Operating Process Map (Minimum Viable)

For any idea that delivers recurring value:

| Process | Purpose | Owner | Key Input | Key Output | Leading Metric | Failure Mode |
|---------|---------|-------|-----------|------------|----------------|--------------|
| Attract | Generate qualified demand | | | | | |
| Convert | Turn demand into revenue | | | | | |
| Deliver | Create and hand off the value | | | | | |
| Retain / Expand | Keep and grow customers | | | | | |
| Improve | Capture learning and raise quality/speed | | | | | |

Fill only the rows that matter for the current stage.

## 2. 30/60/90 Execution Skeleton

- **Days 1-30 (Prove)**: Isolate the single riskiest assumption. Design the smallest test. Define kill/continue criteria. Instrument the metric.
- **Days 31-60 (Stabilize)**: Turn the working experiment into a repeatable process. Document the playbook. Identify the first bottleneck.
- **Days 61-90 (Leverage)**: Systematize or hand off. Introduce the next highest-leverage process. Review unit economics against original targets.

## 3. Decision Filter (CEO Mode)

Before committing resources, force these questions:

1. What is the irreversible decision here versus the reversible one?
2. What is the actual scarce resource right now (cash, time, attention, skill, distribution)?
3. If this works 10x, what breaks first?
4. What would have to be true for this idea to be a bad idea?
5. What is the highest-leverage action that can be taken in the next 48 hours?

## 4. Unit Economics Quick Check

Required minimum fields for any revenue-generating idea:

- Price / AOV
- Gross margin after direct costs
- Customer acquisition cost (CAC)
- Payback period in months
- Expected lifetime value or retention curve
- Contribution margin after variable costs

Flag any model where payback > 12 months or contribution margin is thin without a clear path to scale.
